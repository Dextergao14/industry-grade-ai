- 岗位拆解:Quant Researcher/Trader/Dev/ML,各考什么(本书聚焦 ML+工程,声明概率谜题类自行刷 green book)
- 收益从哪来:alpha/beta、频率光谱(高频→中低频)对 ML 技术栈的影响
- 市场数据:tick/orderbook/K线,数据的脏与偏

### 第 26 章 ML 在量化中的应用 🔸(约 20 页)
- 因子思维:比方"选股=给学生打分,因子=各科成绩";因子挖掘→组合的流水线
- 金融 ML 的特殊性 🔸(和普通 ML 面试区分度最大的部分):信噪比极低、非平稳、样本不独立;为什么 Kaggle 冠军方案在市场上会亏钱
- 时序交叉验证 🔸:泄漏的十种方式(前视偏差/幸存者偏差/标签泄漏),purged K-fold;比方"考试不能提前看到答案,也不能用明年的教材"
- 模型选型现实:为什么树模型(GBDT)仍是主力,深度模型(时序 transformer)用在哪
- 回测 🔸:过拟合回测的骗局(多重检验)、交易成本/滑点建模、💻 写一个含成本的向量化回测框架(~200 行)
- 💻 用公开数据做一个横截面因子模型:特征→purged CV→回测,全流程
- ❓ "怎么判断一个因子是真 alpha 还是过拟合?" "金融数据做 CV 和普通 ML 有什么不同?"

### 第 27 章 量化系统工程速览 🔹→🔸(约 12 页)
- 交易系统链路:行情→信号→风控→执行,延迟预算逐段拆
- 低延迟工程精讲 🔸(SDE 向考点):内核旁路/无锁队列/缓存友好,只讲思想与比方("F1 换胎:每个动作都被压缩到极限"),配 C++ 伪码不展开实现
- Python 侧性能:向量化/numba/polars,💻 把一个慢回测提速 100 倍的案例
- ❓ "为什么高频系统不用 Python?" "怎么给交易系统做延迟优化?"
